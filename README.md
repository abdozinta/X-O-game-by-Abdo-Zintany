# لعبة إكس-أو (Tic Tac Toe)

**مشروع جامعي — تطبيق وحدة تحكم (Console Application) بلغة C (معيار C99)**

---

تم تطوير هذا المشروع وفق **منهجية Agile (المنهجية الرشيقة)**، من خلال **ثمانية Sprints** متتابعة، حيث أُنجز في كل Sprint جزء عملي قابل للاختبار قبل الانتقال إلى المرحلة التالية. يهدف هذا الأسلوب إلى بناء المشروع تدريجياً، مع فصل المسؤوليات، وتسهيل المراجعة الأكاديمية لكل مرحلة على حدة.

---

## وصف المشروع

لعبة **إكس-أو (Tic Tac Toe)** هي تطبيق وحدة تحكم مكتوب بلغة **C** وفق معيار **C99**، يُظهر تطبيق **البرمجة المعيارية (Modular Programming)** ومبادئ **الهندسة النظيفة (Clean Architecture)**.

يلعب لاعبان بالتناوب على شبكة **3×3**. يبدأ **اللاعب X** دائماً، يليه **اللاعب O**. يُدخل كل لاعب رقم الصف والعمود (من 1 إلى 3) لوضع علامته. يقوم البرنامج بـ:

- التحقق من صحة جميع المدخلات
- منع الكتابة فوق الخلايا المشغولة
- اكتشاف جميع **ثمانية** تركيبات الفوز
- التعرف على حالة التعادل
- الإعلان عن النتيجة
- تحديث **لوحة النقاط والإحصائيات** (Scoreboard) للجلسة
- السماح بإعادة اللعب دون إغلاق البرنامج

صُمّم الكود ليكون واضحاً للمراجعة الأكاديمية: دوال صغيرة ومركّزة، أسماء وصفية، وتعليقات تشرح المنطق والغرض من كل جزء.

| البند | التفاصيل |
|-------|----------|
| **اسم المشروع** | Tic Tac Toe Game |
| **اللغة** | C (C99) |
| **نوع التطبيق** | Console Application |
| **أسلوب البرمجة** | Modular Programming |
| **منهجية التطوير** | Agile Methodology |

## طريقة الترجمة (Compilation)

افتح Terminal داخل مجلد `tic_tac_toe` ونفّذ الأمر التالي:

```bash
gcc -Wall -Wextra -std=c99 -o tic_tac_toe main.c game.c utils.c
```

**على Windows (MinGW-w64 / MSYS2):**

```bash
gcc -Wall -Wextra -std=c99 -o tic_tac_toe.exe main.c game.c utils.c
```

### شرح خيارات المترجم

| الخيار | الغرض |
|--------|--------|
| `-Wall -Wextra` | تفعيل تحذيرات المترجم المفيدة لاكتشاف الأخطاء |
| `-std=c99` | استخدام معيار لغة C99 |
| `-o tic_tac_toe` | تحديد اسم الملف التنفيذي الناتج |

## القرارات المعمارية (Architecture Decisions)

قُسّم المشروع إلى **ثلاث وحدات** مستقلة لتحقيق فصل واضح للمسؤوليات (Separation of Concerns):

| القرار المعماري | المبرر |
|-----------------|--------|
| **ثلاث وحدات** (`main`, `game`, `utils`) | فصل تدفق التطبيق عن قواعد اللعبة عن مساعدات الإدخال |
| **هيكل `Board`** | تجميع بيانات اللوحة في كيان واحد يسهل تمريره بين الدوال |
| **تعداد `GameStatus`** | أوضح من استخدام أرقام سحرية (Magic Numbers) لحالات الفوز والتعادل |
| **Scoreboard في `main.c`** | تجنّب المتغيرات العامة؛ حالة الجلسة تُدار على مستوى التطبيق |
| **إدخال 1-based ومصفوفات 0-based** | يتوافق مع طريقة عدّ المستخدم للصفوف والأعمدة على اللوحة |
| **دالة مساعدة `cells_match()`** | تبسّط فحص خطوط الفوز دون إظهار تفاصيل داخلية |
| **وحدة `utils`** | عزل التحقق من الإدخال ليكون قابلاً لإعادة الاستخدام والاختبار |

### مخطط العلاقة بين الوحدات

```
┌─────────────┐
│   main.c    │  ← نقطة الدخول، Scoreboard، حلقة إعادة اللعب
└──────┬──────┘
       │ يستدعي
       ▼
┌─────────────┐     ┌─────────────┐
│   game.c    │────▶│   utils.c   │
│  (game.h)   │     │  (utils.h)  │
└─────────────┘     └─────────────┘
  قواعد اللعبة         إدخال آمن
```

## سير عمل البرنامج

```
Program starts
      ↓
Display welcome message
      ↓
Initialize empty board (inside play_game)
      ↓
Player X turn
      ↓
Display board
      ↓
Get row and column → validate → retry if invalid
      ↓
Place mark
      ↓
Check winner / draw
      ↓
If not finished → switch_player → next turn
      ↓
Display final board
      ↓
Announce result (win or "Draw Game!")
      ↓
Update scoreboard
      ↓
Display scoreboard
      ↓
Ask "Play Again? (Y/N)"
      ↓
Yes → new game  |  No → goodbye and exit
```

### ملخص سير العمل بالعربية

1. **بدء البرنامج** — عرض رسالة الترحيب
2. **تهيئة اللوحة** — داخل `play_game()` تُفرغ جميع الخلايا
3. **دور اللاعب X** — ثم التناوب مع O عبر `switch_player()`
4. **كل دور** — عرض اللوحة → قراءة الإدخال → التحقق → وضع العلامة
5. **بعد كل حركة** — `check_winner()` للفوز أو التعادل
6. **انتهاء المباراة** — عرض اللوحة النهائية والنتيجة
7. **Scoreboard** — تحديث ثم عرض الإحصائيات
8. **إعادة اللعب** — السؤال `Play Again? (Y/N)`؛ نعم = مباراة جديدة، لا = خروج

## مراحل التطوير باستخدام Agile

طُوِّر المشروع على **8 Sprints** متتالية؛ كل Sprint ينتج **Increment** قابلاً للاختبار:

| Sprint | المحور | المخرجات (Deliverable) |
|--------|--------|------------------------|
| **Sprint 1** | إعداد المشروع | هيكل قابل للتجميع، ثوابت، `Board` |
| **Sprint 2** | نظام اللوحة | `init_board()`, `display_board()` |
| **Sprint 3** | إدخال المستخدم | `read_integer_in_range()`، إدخال آمن (1–3) |
| **Sprint 4** | آليات اللعب | `make_move()`, `is_valid_move()`, `switch_player()` |
| **Sprint 5** | التحقق من الفوز | `check_winner()` — 8 خطوط فوز |
| **Sprint 6** | التحقق من التعادل | `is_board_full()`، رسالة `Draw Game!` |
| **Sprint 7** | تجربة المستخدم | ترحيب، نتائج، `ask_play_again()` |
| **Sprint 8** | نظام النقاط | `Scoreboard`، `update_scoreboard()`, `display_scoreboard()` |

---

### Sprint 1 — إعداد المشروع

**المهام:**
- إنشاء هيكل المجلدات والملفات
- تعريف الثوابت (`BOARD_SIZE`, `EMPTY_CELL`, `PLAYER_X`, `PLAYER_O`)
- تعريف هيكل `Board` وتعداد `GameStatus`
- إعلانات الدوال في `game.h` و `utils.h`

**المخرج:** هيكل مشروع **قابل للتجميع (Compilable Skeleton)**

---

### Sprint 2 — نظام اللوحة

**المهام:**
- تنفيذ `init_board()` لتهيئة اللوحة
- تنفيذ `display_board()` لعرض الشبكة المرئية

**المخرج:** لوحة 3×3 تُعرض بشكل صحيح مع تسميات الصفوف والأعمدة

---

### Sprint 3 — إدخال المستخدم

**المهام:**
- قراءة الصف والعمود بأمان
- التحقق من النطاق (1–3 فقط)
- رفض الحروف والرموز

**المخرج:** نظام إدخال آمن عبر `utils.c` و `get_player_move()`

---

### Sprint 4 — آليات اللعب

**المهام:**
- وضع العلامة عبر `make_move()`
- منع الكتابة فوق خلايا مشغولة
- التناوب التلقائي: X → O → X → O

**المخرج:** نظام أدوار يعمل بشكل كامل

---

### Sprint 5 — التحقق من الفوز

**المهام:** فحص جميع تركيبات الفوز:

| النوع | العدد |
|-------|-------|
| صفوف (Rows) | 3 |
| أعمدة (Columns) | 3 |
| أقطار (Diagonals) | 2 |
| **المجموع** | **8** |

**المخرج:** اكتشاف الفائز بشكل صحيح عبر `check_winner()`

---

### Sprint 6 — التحقق من التعادل

**المهام:**
- اكتشاف امتلاء اللوحة عبر `is_board_full()`
- إرجاع `GAME_DRAW` عند عدم وجود فائز

**المخرج:** رسالة **Draw Game!** عند التعادل

---

### Sprint 7 — تجربة المستخدم

**المهام:**
- شاشة ترحيب في `main.c`
- رسائل النتيجة (فوز / تعادل)
- خيار إعادة اللعب: `Play Again? (Y/N)`

**المخرج:** لعبة كاملة قابلة للعب من البداية إلى النهاية

---

### Sprint 8 — نظام النقاط والإحصائيات

**المهام:**
- تتبع انتصارات X و O والتعادلات
- `update_scoreboard()` و `display_scoreboard()`
- استمرار الإحصائيات طوال الجلسة (تُصفّر عند إغلاق البرنامج فقط)

**المخرج:** نظام إحصائيات الجلسة (Session Statistics)
